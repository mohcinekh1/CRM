# 📌 CONTEXT — Smart CRM Project

## Vue d'ensemble du projet
Smart CRM est une application de gestion de la relation client avec scoring automatique, segmentation, campagnes email et génération de contenu par IA locale (Ollama).

## Stack technique
| Couche        | Technologie                        |
|---------------|------------------------------------|
| Backend       | Spring Boot 3.x + Java 21          |
| Frontend      | Angular 20                         |
| Base de données | MySQL 8.x                        |
| IA / LLM      | Ollama (Mistral) — local           |
| Email         | Spring Mail + Gmail SMTP           |
| ORM           | Spring Data JPA + Hibernate        |
| Mapping       | MapStruct                          |
| Sécurité      | Spring Security + JWT              |
| Build Backend | Maven                              |
| Build Frontend | npm / Angular CLI                 |

## Entités principales
- **Customer** → le cœur du CRM (id, firstName, lastName, email, phone, address, customerState, createdAt)
- **Interaction** → historique des contacts (APPEL, EMAIL, REUNION, NOTE)
- **Segment** → regroupement de clients par critères
- **CustomerSegment** → table pivot Many-to-Many
- **Campaign** → campagnes email ciblées par segment
- **EmailLog** → trace de chaque email envoyé
- **CustomerScore** → score calculé automatiquement (FIDELE / A_RISQUE)

## Relations clés
- Customer 1 → * Interaction
- Customer * → * Segment (via CustomerSegment)
- Customer 1 → * CustomerScore
- Customer 1 → * EmailLog
- Segment 1 → * Campaign

## Rôles utilisateurs
- **ROLE_ADMIN** → accès complet (segments, campagnes, dashboard global)
- **ROLE_COMMERCIAL** → gestion clients, interactions, lecture scores

## Convention de nommage
- **Entités JPA** → PascalCase, suffixe aucun (ex: `Customer`)
- **DTOs** → PascalCase + suffixe DTO (ex: `CustomerDTO`, `CustomerCreateDTO`)
- **Services** → PascalCase + suffixe Service (ex: `CustomerService`)
- **Controllers** → PascalCase + suffixe RestController (ex: `CustomerRestController`)
- **Endpoints REST** → `/api/{ressource}` en kebab-case
- **Composants Angular** → kebab-case (ex: `customer-list`, `customer-detail`)

## Règles de développement
1. Ne jamais exposer les entités JPA directement dans les réponses REST → toujours passer par un DTO
2. Toujours valider les DTOs avec `@Valid` côté backend
3. Chaque service doit avoir sa propre responsabilité (Single Responsibility)
4. Les exceptions doivent être gérées centralement via `@ControllerAdvice`
5. Toute modification de BDD passe par `database/schema.sql` (et jeux de données dans `database/data.sql`)
6. Le frontend ne stocke que le JWT (localStorage), jamais le mot de passe
